# Canonical train/eval/test splits

`train_task_ids.txt` (4000) / `eval_task_ids.txt` (1000) / `test_task_ids.txt` (1000).

Drawn from the 29,996-task pool for which Gemma-3-27B-IT activations already exist at
`activations/gemma-3-27b_it/pref_main/`. Disjoint from each other and from all prior
experiment exclude-lists (main 10k training pool, 4k eval pool, 3k run, gpt-oss campaigns,
mra_exp2 splits a/b/c).

Stratified by **dataset** (stresstest, competition_math, alpaca, wildchat) and **topic**
(14-way from `data/topics/topics.json`). Dataset proportions target 40 / 20 / 20 / 20 %
but wildchat is capped at its total availability (757 tasks), so stresstest was topped up
to make the splits hit exactly 4000 / 1000 / 1000.

Actual per-split composition:

```
              train  eval  test
stresstest     1895   474   474   (47%)
competition_   800    200   200   (20%)  math
alpaca         800    200   200   (20%)
wildchat       505    126   126   (13%)
```

Topic distribution (top by train count): math 22%, stresstest_other 21%, knowledge_qa 12%,
harmful_request 10%, content_generation 9%, value_conflict 8%, and smaller tails for
fiction, security_legal, coding, etc.

## Use

All cross-persona and cross-prompt experiments should pull their task IDs from these
three files so results are directly comparable:

- **train**: probe training (Ridge, Bradley-Terry). Already has baseline activations + Thurstonian μ via the main 10k measurements (existing pool covers these task_ids as a subset when filtered).
- **eval**: α-selection + heldout r.
- **test**: the cross-persona evaluation set — measurements under each new persona's system prompt go here.

## Reproduction

Generated by `scripts/canonical_splits/make_splits.py`. Seed = 42. To regenerate
(deterministic), re-run that script.
