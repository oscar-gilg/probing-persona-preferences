# Qwen-3.5-122B Persona Transfer — Extraction + Utilities

First step of the Qwen replication of the Gemma persona probe-transfer result
(`experiments/persona_sweep/probe_transfer/probe_transfer_report.md`). Replicates
the extraction + measurement phase (Gemma's
`experiments/persona_sweep/extraction/extraction_spec.md` and the final-six AL
measurement configs) on Qwen-3.5-122B.

Once complete, per-persona probes + the 7×7 transfer / utility / cosine
matrices are produced by a separate follow-up spec
(`probe_transfer_spec.md`, to be written after this step finishes).

## Personas

Same list as the Gemma replication:

- `default` (no persona; extraction has no system prompt, measurement uses
  `qwen3.5-122b-nothink` which runtime-injects `/no_think`)
- `final_six`: `sadist, mathematician, aura, strategist, contrarian, slacker`

Source of truth for prompts: `experiments/persona_sweep/sweep_personas.json`
(`metadata.final_six` + `personas[]`). Persona prompts are model-agnostic and
reused byte-for-byte from Gemma.

## Pod provisioning

Launch via `/zombuul:launch-runpod qwen-persona-transfer --disk-gb 500 --volume-gb 30`.
Pick **3× NVIDIA A100-SXM4-80GB** from the GPU options — the MoE needs
> 160 GB VRAM; 3 cards fit bf16 cleanly without CPU offload.

Disk/volume rationale (covered generally in zombuul's `run-experiment`
sizing guide):

- `--disk-gb 500`: container disk holds the ~244 GB Qwen-3.5-122B bf16 HF
  cache + venv + headroom. Regenerable; wiped on pause.
- `--volume-gb 30`: MooseFS volume holds the cloned repo + ~3 GB of
  extraction outputs. Keep small to stay well under the per-user quota.

Environment exports (set before any python invocation on the pod):

- `HF_HUB_DISABLE_XET=1` — xet's concurrent-small-writes pattern is
  unreliable on MooseFS-adjacent storage; forces legacy single-stream download.
- `HF_HUB_ENABLE_HF_TRANSFER=0` — same reason.

Pod setup leaves a few deps uninstalled for our use case — on first boot:
`pip install -e ".[dev]"` and `pip install --upgrade "transformers @ git+https://github.com/huggingface/transformers.git"` (transformers' `qwen3_5_moe` support
lives on main, newer than any pypi release) and `apt-get install -y tmux`.

## Task set — reuse Gemma's canonical splits

Reuse `data/canonical_splits/{train,eval,test}_task_ids.txt` unchanged
(4,000 / 1,000 / 1,000 = 6,000 tasks), same file paths Gemma used. This
gives us task-level alignment: per-task Qwen vs Gemma utilities,
probe-scores, and direct Qwen↔Gemma comparisons on the same task ids.

The cost is re-extraction: only 1,655 / 6,000 canonical tasks are in
Qwen's existing 14,038-task pool (`qwen35_all_task_ids.txt`), so the
existing `activations/qwen35_122b/pref_main/` cannot be reused even for
`default`. All 7 personas (including `default`) need fresh extraction on
the canonical 6,000.

No new split-generation script needed; no new canonical-splits directory.

## Activation extraction

Seven extraction configs (one per persona, `default` included) at
`configs/extraction/qwen35_pref_<persona>_sweep.yaml`, generated by
`scripts/persona_sweep_extraction/gen_qwen_configs.py` (adaptation of
`gen_configs.py` — same persona-prompt source, different model and output
path, adds `default` as a 7th config with an empty `system_prompt`).

All 7 share (pruned vs the existing Qwen sweeps — we already know tb-1/tb-4
at L38 are the working probe selectors):

- `model: qwen3.5-122b` (NOT the `-nothink` variant; extraction runs without
  `/no_think` injection, matching the existing `pref_main`)
- `selectors: [turn_boundary:-1, turn_boundary:-4]` — the two probe selectors
- `layers_to_extract: [33, 38, 43]` — L38 probe + one neighbour either side
- `task_ids_file: data/canonical_splits/all_6000_task_ids.txt`
- `task_origins: [wildchat, alpaca, math, bailbench, stress_test]`
- `batch_size: 4` (matches the E1c-proven Qwen extraction; MoE is memory-tight)
- `max_new_tokens: 1` (skip generation; persona-transfer analysis uses
  activations only)
- `save_every: 200`, `device: auto`, `temperature: 1.0`, `seed: 42`
- `output_dir: /workspace/activations/qwen35_122b/pref_<persona>_sweep`

Outputs write directly to `/workspace/` (MooseFS, durable across pause).
Total write volume across all 7 personas is ~3 GB — small enough to stay
well under the per-user quota and small-enough-per-file to avoid the
I/O-error patterns that only show up at HF-cache scale.

Per-persona field: `system_prompt` — the persona's prompt for the final
six, or unset for `default`.

Orchestration: `scripts/persona_sweep_extraction/run_qwen_all.sh` runs the 7
configs sequentially, logging to `/workspace/logs/extract_qwen_<persona>_sweep.log`.

### GPU requirements

3× A100-SXM4-80GB via `device_map=auto`. The 122B MoE at bf16 totals ~244 GB
parameters — 2 cards force CPU offload which slows extraction substantially.
3 cards fit the model cleanly in VRAM.

6000 tasks × prompt encoding only (`max_new_tokens=1`) × batch 4 → ~55 min
per persona, ~6.5 h total for the 7. First persona adds ~30 min of HF
cache download.

### Expected output

Per persona: ~6000 × 2 selectors × 3 layers × bf16 ≈ 0.5 GB. Total for
7 personas: ~3.5 GB.

## Utility measurement

### 21 AL configs (7 personas × 3 splits)

One per `(persona, split)` at
`configs/measurement/qwen_persona_sweep/final_six/<persona>_<split>.yaml`,
generated by `scripts/persona_sweep_extraction/gen_qwen_measurement_configs.py`
(adaptation of `gen_measurement_configs.py`).

Fixed fields (match Gemma persona-sweep measurement configs, adjusted for Qwen):

- `preference_mode: pre_task_active_learning`
- `model: qwen3.5-122b-nothink` (matches probe training + all existing Qwen
  AL runs)
- `temperature: 1.0`
- `task_origins: [wildchat, alpaca, math, bailbench, stress_test]`
- `stratified_sampling: false`
- `task_sampling_seed: 42`
- `templates: src/measurement/elicitation/prompt_templates/data/completion_preference.yaml`
- `response_formats: [completion]`
- `n_samples: 3`
- `pair_order_seed: 42`
- `active_learning`: `initial_degree: 3, max_iterations: 10, p_threshold: 0.3,
  q_threshold: 0.3, convergence_threshold: 0.98, seed: 42` (Gemma used
  `initial_degree: 2`; we bump to 3 for a richer seed phase)

Per-split fields:

| Split | `n_tasks` | `include_task_ids_file` | AL `batch_size` |
|---|---:|---|---:|
| train | 4000 | `data/canonical_splits/train_task_ids.txt` | 1200 |
| eval | 1000 | `data/canonical_splits/eval_task_ids.txt`  | 300 |
| test | 1000 | `data/canonical_splits/test_task_ids.txt`  | 300 |

Per-persona fields:

- 6 non-default: `measurement_system_prompt: <persona prompt>` and
  `experiment_id: qwen_persona_sweep_final_six_<persona>_<split>`
- `default`: NO `measurement_system_prompt` (the `-nothink` model auto-injects
  `/no_think` — mirrors the existing `qwen35_10k_active_learning` setup) and
  `experiment_id: qwen_persona_sweep_final_six_default_<split>`

### Dir conventions

Results land at
`results/experiments/qwen_persona_sweep_final_six/pre_task_active_learning/`
under auto-generated subdirs of the form
`completion_preference_qwen3.5-122b-nothink_completion_canonical_seed0_<sysHASH>_<taskset>`.
Rename / symlink to short names (`<persona>_<split>`) with
`scripts/persona_sweep_extraction/rename_exp_dir.py`
(already used for Gemma's `default_*` runs and should work unchanged).

### Cost and runtime

Scaling from Gemma's actuals (`persona_sweep_final_six/*/active_learning.yaml`):
- Gemma train (init_degree=2): ~33.5k comparisons / run
- Gemma eval/test (init_degree=2): ~8.4k comparisons / run

With `initial_degree: 3` on Qwen, the seed phase grows by ~50 %, pushing:
- train ≈ 38–40k comparisons / run
- eval/test ≈ 10k / run

Per persona: ~58–60k comparisons. Across 7 personas × 3 splits: **~410k total
API calls**, each = 1 Qwen completion + 1 gpt-5-nano judge parse.

Rough budget (OpenRouter, mid estimate): **~$80–160** across all 21 runs.
Qwen dominates (~$70–140); gpt-5-nano judge ~$20. Check balance first.

Run with `--max-concurrent 50`, wrapped in tmux. Wall-clock is dominated by
the longest AL run (train splits); expect several hours end-to-end, not days.

## Execution order

1. **Generate configs** (already checked in — skip if present):
   `python -m scripts.persona_sweep_extraction.gen_qwen_configs` and
   `python -m scripts.persona_sweep_extraction.gen_qwen_measurement_configs`.

2. **Provision pod** per the "Pod provisioning" section above
   (`/zombuul:launch-runpod` with the sizing flags, then `/zombuul:provision-pod`
   to sync `.env` and the spec).

3. **On the pod, install missing deps** (per the "Pod provisioning" section):
   `pip install -e ".[dev]"`, upgrade transformers from git, `apt-get install -y tmux`.

4. **Launch extraction in a tmux session** so it survives SSH disconnects:

   ```bash
   tmux new -d -s extraction "set -a; source .env; set +a; \
     export HF_HUB_DISABLE_XET=1; export HF_HUB_ENABLE_HF_TRANSFER=0; \
     bash scripts/persona_sweep_extraction/run_qwen_all.sh 2>&1 \
     | tee /workspace/logs/run_qwen_all.log"
   ```

5. **Attach a babysit** so the main session can do other work:
   `/zombuul:babysit qwen-persona-transfer "Qwen persona-sweep extraction — 7 personas writing to /workspace/activations/qwen35_122b/pref_<persona>_sweep/. Done when 14 NPZs present and log ends with '=== done: slacker ==='."`

6. **In parallel on the laptop**, launch the 21 AL runs in a durable tmux:

   ```bash
   export SSL_CERT_FILE=$(python -m certifi); export REQUESTS_CA_BUNDLE=$SSL_CERT_FILE; \
   python -m src.measurement.runners.run \
     configs/measurement/qwen_persona_sweep/final_six/*.yaml \
     --max-concurrent 50 \
     --experiment-id qwen_persona_sweep_final_six
   ```

   The SSL cert exports sidestep a ~30 min startup hang in
   `ssl.SSLContext.load_verify_locations` under many concurrent clients.

7. **When extraction completes** (log ends with `=== done: slacker ===` and
   all 7 `pref_<persona>_sweep/` dirs have both NPZs), rsync activations to
   the laptop:

   ```bash
   rsync -az runpod-qwen-persona-transfer:/workspace/activations/qwen35_122b/ activations/qwen35_122b/
   ```

8. **When AL completes**, rename the auto-generated result subdirs to
   `<persona>_<split>` via `scripts/persona_sweep_extraction/rename_exp_dir.py`.

9. **Pause the pod** via `/zombuul:pause-runpod`. Outputs are already on
   `/workspace/` (preserved) and mirrored on the laptop; only HF cache and
   venv are lost — both regenerable if we resume.

## Sanity checks before handoff

Before declaring the phase done:

1. **Activation completeness.** Each of the 7 `pref_<persona>_sweep/`
   directories contains 2 `activations_turn_boundary:-N.npz` files
   (`-1` and `-4`); every NPZ has all 6000 canonical task IDs and all
   3 layers (`[33, 38, 43]`).
2. **Utility completeness.** All 21 AL runs have
   `converged: true` (or at worst `pair_agreement >= 0.9` with
   `aborted_api_failures: false`); Thurstonian fits exist
   (`thurstonian_*.csv`) for every `(persona, split)`.
3. **Model alignment.** Every AL `config.yaml` shows
   `model_short: qwen3.5-122b-nothink`; every extraction
   `extraction_metadata.json` shows `model: qwen3.5-122b` and the persona's
   `system_prompt` (or `null` for `default`).
4. **Refusal rates.** Under `sadist` the refusal rate will be high; log but
   don't block. Flag any other persona with >10 % refusals for inspection.

## Deviations from Gemma, for the record

- **Model + runtime behaviour.** `qwen3.5-122b-nothink` for measurement
  auto-injects `/no_think`; `qwen3.5-122b` for extraction does not. The
  Gemma equivalent was plain `gemma-3-27b` everywhere.
- **Default persona.** Gemma default = empty system prompt for both
  extraction and measurement. Qwen default = empty system prompt for
  extraction (matching `pref_main`) and runtime-injected `/no_think` for
  measurement (matching `qwen35_10k_active_learning`). This mismatch exists
  in the existing Qwen probe setup and is preserved here.
- **Tokenizer / selectors.** Qwen uses the 5-token turn boundary
  (`<|im_end|>\n<|im_start|>assistant\n`); tb-1 and tb-4 are the established
  probe-training selectors for Qwen (vs tb-2/tb-5 for Gemma). Only these
  two are extracted — the wider sweep already lives in
  `activations/qwen35_122b/pref_main/` (training-probes report).
- **Layers.** Qwen probes sit at layer 38; extraction covers `[33, 38, 43]`
  (L38 + nearest-tested neighbours from the original probe sweep). Enough
  for a lean layer-dependence figure; add more later if we need resolution.
- **AL initial_degree.** Bumped from Gemma's 2 → 3 for a richer seed phase.
- **Extraction efficiency.** `batch_size: 4`, `max_new_tokens: 1` (no
  generation; we only need pre-CoT activations). Gemma ran `batch_size: 32`,
  `max_new_tokens: 512`.
