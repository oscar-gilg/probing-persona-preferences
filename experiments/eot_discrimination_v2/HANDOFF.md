# EoT discrimination v2 — handoff

Brief pointer doc for re-running §3.1 (Cohen's d figures) with new sysprompts or fine-tuned personas (e.g. SFT'd Qwen evil persona). The full experiment lives on branch `eot_discrimination_v2`; this file lives on `main` so it's discoverable from any branch.

## What v2 is

Drop-in replacement of §3.1 stimuli with high-quality 1000-item-per-domain corpora. CIs on Cohen's d added to all 5 §3.1 figure scripts. Pipeline (probes / layers / scoring infra / token convention) is unchanged from v1.

## Where stuff lives

All paths relative to repo root. Everything is on branch `eot_discrimination_v2` unless noted.

### Build scripts

- `experiments/eot_discrimination_v2/scripts/build_truth.py` — intersects `data/creak/known_correct_*.json` (per-model knowledge filter from `scripts/filter_creak.py`) and writes truth_v2.
- `experiments/eot_discrimination_v2/scripts/build_harm.py` — BailBench → LLM-paired benign + 2-of-3 LLM-judge filter → harm_v2.
- `experiments/eot_discrimination_v2/scripts/build_politics.py` — OpinionQA stance translation + 3-of-3 LLM-judge filter → politics_v2.
- `experiments/eot_discrimination_v2/scripts/build_assistant_turns.py` — adds `messages_assistant` field to each corpus (LLM-generates the user question or assistant response per item).

### Scoring infra (the bit you'll reuse)

- `experiments/eot_discrimination_v2/scripts/build_scoring_inputs.py` — crosses corpora × sysprompts × turn placement → `scoring_inputs/{user|assistant}_turn.json` (flat list of records ready for `score_prompt_batch`).
- `experiments/eot_discrimination_v2/scripts/run_scoring.py` — loads target model + probes, runs `score_prompt_batch` over scoring inputs, writes `scoring/{model}/{user_turn_,}scoring_results.json`.
- `experiments/eot_discrimination_v2/scripts/run_all_scoring.sh` — chains all 4 cells (Gemma user / Gemma asst / Qwen user / Qwen asst).

### Token convention (READ CAREFULLY)

`turn_boundary:N` is anchored at `first_completion_index` — see `src/models/base.py:108-110` and the paper's App. token-selection figure. **`turn_boundary:-1` is NOT eot in general.** This run scores at `scores_arr[-1]` of `score_prompt_all_tokens` with `add_generation_prompt=False`. Under that convention:

- Gemma chat template ends with `...<end_of_turn>\n` → `tokens[-1] = '\n'` (the eot-adjacent newline; the residual stream there carries information through `<end_of_turn>`).
- Qwen chat template ends with `...<|im_end|>\n` → same pattern.
- v1 used the same convention (named the field `eot_scores` — slight misnomer; v2 calls it `probe_scores`).
- Pilot in `run_scoring.py` decodes `tokens[-5:]` and asserts the eot literal appears in the last 5 tokens.

### v2 baseline numbers (for comparison after a re-run)

See `experiments/eot_discrimination_v2/eot_discrimination_v2_report.md` for the full Cohen's d table. Key cells (assistant-turn neutral baseline):
- Gemma harm neutral: d = −3.83 [−4.04, −3.62]
- Gemma truth neutral: d = +1.11 [+0.97, +1.24]
- Qwen harm neutral: d = −2.62 [−2.79, −2.45]
- Qwen truth neutral: d = +1.47 [+1.33, +1.61]

Open follow-up flagged in v2 report: **Qwen lying personas at user-turn no longer flip the sign on truth** (in v1 they did); prompted-persona effects on Qwen are weaker than v1 suggested. This is exactly the motivation for the SFT evil-persona variant.

## How to re-run with a new persona

For an SFT'd Qwen evil-persona model, the re-run is mostly a model-swap + sysprompt-swap:

1. **Swap the model handle** in `experiments/eot_discrimination_v2/scripts/run_scoring.py:PROBE_SETS_BY_MODEL` and `LAYERS_BY_MODEL` and `EXPECTED_EOT_BY_MODEL`. Add a new key for the SFT model (or reuse the existing one if the chat template is unchanged).
2. **Swap the sysprompt set** in `build_scoring_inputs.py:SYSPROMPTS_BY_DOMAIN`. For an SFT evil persona, the sysprompt is whatever you trained it to respond to (could be `neutral` if the persona is baked in, or a custom marker). The `messages` and `messages_assistant` from the corpus are reused as-is.
3. Re-run `build_scoring_inputs.py` (CPU, local) → push to a pod.
4. Re-run `run_all_scoring.sh` on a 4×A100 pod (~1h wall-clock for ~31k forward passes).
5. Compare to the v2 baseline numbers above.

If the SFT changes the tokenizer, also re-pilot the eot check (the pilot in `run_scoring.py` will catch it).

## Why this exists

§3.1 v1 had 88-stimulus truth / 77-pair harm / 234-stimulus politics → wide CIs. v2 scaled stimuli ~10× with multi-LLM-agreement quality filters. The result: CIs ~3× tighter, but several v1 d magnitudes shifted (some up, some down) — which exposed v1's surface-form artefacts and surfaced the Qwen-prompted-persona weakness mentioned above.
